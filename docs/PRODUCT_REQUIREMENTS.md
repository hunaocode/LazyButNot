# 懒人不懒 iOS App 产品需求文档

## 1. 文档说明

### 1.1 产品名称
- 中文名：懒人不懒
- 英文名：LazyButNot

### 1.2 文档目的
本文件用于固化当前本地工程的真实产品状态，作为后续迭代、联调和验收基线。

说明：
- 本文档以当前代码实现为准，不再沿用早期的理想化 MVP 设想。
- 未落地功能会单独列在“待迭代”中，不与已实现功能混写。

### 1.3 产品定位
一款面向个人使用的 iOS 原生目标坚持 App。核心目标不是强调高压自律，而是帮助用户把长期目标拆成最小动作，通过本地记录、周期提醒、截止监督和倒计时闹钟，尽量减少中断。

## 2. 产品背景与设计原则

用户在长期目标执行中，常见问题不是“不想做”，而是：
- 目标太大，开始成本高
- 当下没有外部触发，容易拖延
- 长期目标反馈慢，短期很难形成正向回路
- 状态差时一旦彻底断掉，就很难恢复

因此本产品遵循以下原则：
- 目标必须可执行，强调“最小动作”
- 反馈必须短周期可见，优先解决“今天该做什么”
- 提醒应主动触发，而不是依赖用户自己记得
- 允许“保底完成”，避免一次中断导致整体放弃
- 所有目标独立管理，避免互相干扰
- 默认以本地能力完成，不依赖服务端

## 4. 平台兼容与提醒策略

### 4.1 系统兼容策略
- iOS 17-25：使用 `UserNotifications` 提供普通通知提醒与单次通知音
- iOS 26 及以上：在普通通知之外，支持 `AlarmKit` 闹钟式提醒与倒计时 Live Activity

### 4.2 当前提醒模式
系统当前存在三种提醒状态：
- 普通通知模式：仅通知提醒
- 闹钟未授权：设备支持 AlarmKit，但用户尚未授权，因此只能退回普通通知
- 闹钟模式：AlarmKit 已授权，可使用闹钟式监督提醒和倒计时闹钟

### 4.3 权限申请策略
- App 启动后，进入前台激活阶段会尝试申请通知权限
- 在 iOS 26 及以上，若系统支持且权限状态为未决定，也会尝试申请 AlarmKit 权限
- 设置页可以再次触发权限申请或跳转系统设置
- 权限状态会在设置页中明确展示
- 在新建/编辑目标页中，只有“监督提醒已开启”时才允许进一步开启“闹钟式提醒”
- 目标页中的“闹钟式提醒”默认仅在 iOS 26+ 且 AlarmKit 已授权时为开启状态，否则默认关闭
- 当用户在目标页主动打开“闹钟式提醒”时：
- 若系统低于 iOS 26，会提示升级系统
- 若权限被拒绝，会提示前往系统设置
- 若权限为未决定，会立即触发 AlarmKit 授权请求

### 4.4 当前提醒能力边界
- 目标提醒当前统一采用“固定日期预设窗口”策略，而不是长期重复规则
- 普通提醒会按目标周期预先注册未来一段时间内的固定日期本地通知
- 截止前监督提醒会根据目标配置的提前量，为窗口内每个目标日期逐条生成本地通知或 AlarmKit 闹钟
- 若目标开启“闹钟式提醒”且设备为 iOS 26+、且已获得 AlarmKit 授权，则截止监督优先走 AlarmKit
- 若 AlarmKit 不可用或未授权，则回退为普通通知提醒
- 通知前台展示已处理，前台收到通知时会展示横幅并携带声音
- 当前预设窗口为 7 天
- 对“每周固定日”目标，系统会向后扫描 21 天，并为命中的选中 weekday 生成提醒

## 5. 已实现功能

### 5.1 目标管理
用户可以创建多个独立目标，当前目标字段包括：
- 目标名称
- 目标说明
- 目标分类
- 最小动作
- 周期类型
- 每周次数目标
- 每周固定日
- 提醒时间
- 截止时间
- 是否开启监督提醒
- 监督提前量列表
- 是否开启响铃/闹钟式提醒
- 是否暂停

支持的目标操作：
- 创建目标
- 编辑目标
- 删除目标
- 暂停提醒
- 恢复提醒
- 在编辑时查看当前闹钟式提醒状态

### 5.2 周期类型
当前已支持的周期类型：
- 每日
- 每周固定日
- 每周次数

说明：
- 每周固定日：仅在选中的星期触发“今日待完成”
- 每周次数：以本周累计完成次数为准，达标后本周不再继续视为待完成

### 5.3 打卡机制
当前支持两种有效打卡状态：
- 完成
- 保底完成

说明：
- 两种状态都记为有效完成
- 统计时会区分“连续完成”和“持续坚持”
- 当天重复打卡会更新当日记录，而不是产生多条同日记录

### 5.4 首页
首页当前包含：
- 今日进度摘要卡
- 今日待完成目标
- 今日已完成目标
- 一键启动“专注模式”倒计时闹钟入口

当前首页逻辑：
- 今日列表由“今天应完成的目标”与“今天已完成的目标”共同组成
- 对每周次数目标，若本周刚好在今天达标，仍会显示在“已完成”区，不会从首页直接消失
- 排序按今天截止时间升序
- 摘要卡上的专注入口通过弹层创建倒计时闹钟
- 今日未完成卡片保留“完成 / 保底完成”按钮，并在原地打卡
- 今日已完成或保底完成卡片不再显示操作按钮，而是显示紧凑的“已打卡 / 已保底打卡”状态徽标
- 今日卡片除明确的交互按钮区之外，其他内容区域均可进入目标详情页

### 5.5 目标列表页
目标列表页当前展示：
- 名称
- 暂停状态
- 最小动作
- 分类
- 提醒时间
- 持续坚持天数

支持：
- 进入详情页
- 新建目标
- 左滑删除目标
- 当前目标列表已引入卡片式视觉样式，并跟随全局主题变化
- 目标卡片默认不展示系统样式的小箭头和上下分割线，保留固定页边距和卡片层次

### 5.6 目标详情页
目标详情页当前展示：
- 目标名称与说明
- 关键指标卡
- 规则信息
- 今日操作区
- 历史记录

关键指标展示规则：
- 对“每日 / 每周固定日”目标：显示“连续完成”和“持续坚持”
- 对“每周次数”目标：显示“本周完成 x/y”与“本周状态”

详情页支持操作：
- 标记完成
- 标记保底完成
- 暂停/恢复提醒
- 编辑目标

总览区：

按目标区：

### 5.8 设置页
设置页当前展示：

- 权限申请入口
- 重新同步所有提醒入口
- 主题切换入口
- 产品原则说明

设置页当前承担的能力：
- 查看权限状态
- 请求通知权限
- 请求 AlarmKit 权限
- 在权限被拒绝时跳转系统设置
- 重新根据当前目标全量注册提醒
- 在本地切换当前 App 视觉主题

当前视觉规则：
- 设置页容器背景会跟随当前 App 主题同步变化
- 设置页中的用户可见文案已纳入多语言范围

### 5.9 普通通知提醒
普通通知能力基于 `UNUserNotificationCenter`。

当前行为：
- 每个目标按周期预设固定日期提醒，而不是注册长期重复提醒
- 普通提醒与监督提醒统一使用滚动窗口策略
- 当前普通提醒窗口为 7 天
- 对“每周固定日”目标，会在未来 21 天内筛出命中的星期并逐日预设
- 对“每周次数”目标，会基于当前周是否达标计算未来待提醒日期
- 监督提醒按照截止前的提前量逐条注册
- 普通提醒带默认通知音
- 若监督提醒开启声音，则对应监督通知带单次提示音

### 5.10 文案与本地化
当前产品要求：
- 页面标题、按钮、表单、卡片、状态文案支持中英文
- 通知标题和正文支持中英文
- AlarmKit / 倒计时 / Live Activity / Widget 文案支持中英文
- 主题名、主题标语支持中英文

工程要求：
- 新增或修改用户可见文案后，必须同步更新字符串目录
- 本地化校验应通过脚本自动完成，而不是人工逐页检查
- 当前实现没有自动补窗机制，超出已预设窗口后，需要依赖用户操作、设置页“重新同步所有提醒”或 App 启动首轮调度重新生成未来提醒

### 5.10 AlarmKit 闹钟式提醒
在 iOS 26+ 且已授权时，目标的监督提醒可走 AlarmKit。

当前已实现能力：
- 根据目标截止时间、监督提前量和当前预设窗口生成 AlarmKit 闹钟
- 闹钟触发优先用于“临近截止强提醒”
- 目标被暂停、删除、重新调度时，会同步取消旧的 AlarmKit 任务
- 当用户在当天完成目标后，会先清空该目标当天及未来已挂载的提醒，再从次日开始重建未来窗口，确保当天剩余监督闹钟不再继续触发

当前边界：
- 目前主要用于监督提醒
- 尚未为目标监督提醒接入复杂的自定义 `AppIntent` 行为
- AlarmKit 当前也采用固定日期预设，而不是完全依赖重复型 schedule

### 5.11 倒计时闹钟
首页支持创建独立的专注倒计时闹钟。

当前倒计时闹钟支持：
- 输入提醒标题
- 选择场景分类
- 选择倒计时时长
- 可选再次倒计时
- iOS 26+ 下通过 AlarmKit 调度
- 创建成功后立即进入专注详情页
- 倒计时进行中，专注详情页会实时展示剩余时间
- 专注详情页隐藏导航栏和 Tab Bar，仅保留底部“放弃专注”操作
- 用户点击“放弃专注”时，会取消当前倒计时并返回首页
- 当前实现只允许同时存在一个活动中的倒计时闹钟；新建倒计时时会先取消旧的活动倒计时
- 当前通过 `UserDefaults` 持久化活动中的专注会话，用于 App 冷启动或从系统入口恢复页面状态

### 5.12 Widget Extension 与 Live Activity
当前工程已经包含 Widget Extension，并为倒计时闹钟提供完整的 Live Activity 展示与回跳能力。

当前 Live Activity 已实现：
- 锁屏展示
- 灵动岛展示
- 待机显示倒计时适配
- 根据倒计时状态展示 countdown / paused / alert 三种视图
- 基于场景元数据切换图标、色调和标题
- 点击 Live Activity 或灵动岛，会通过深链进入专注详情页
- App 回到前台时，会同步检查 Live Activity 是否已被用户清除
- 若系统侧 Live Activity 已被清除，App 会主动取消本地记录的活动倒计时，避免“界面消失但闹钟仍继续”的状态不一致

### 5.13 统一分钟选择组件
当前已抽离统一的分钟选择组件，用于所有“分钟数”配置场景。

当前能力包括：
- 支持预设分钟选项
- 支持“自定义”滚轮选择
- 支持展示默认占位文案
- 支持识别当前是否命中自定义值
- 支持清空自定义值

当前接入位置：
- 倒计时时长选择
- 倒计时再次提醒间隔
- 目标监督提前量的自定义选择

### 5.14 颜色主题系统
当前工程已支持 App 级颜色主题切换。

当前能力包括：
- 提供一套默认基础主题
- 提供多套可切换的扩展主题样式
- 主题切换结果本地持久化
- 首页摘要卡、今日目标卡片、目标列表卡片会随主题同步变化
- 统计页与设置页背景会随主题同步变化
- 首页已打卡状态徽标和主要操作按钮会随主题强调色同步变化
- Tab Bar 强调色会随主题同步变化

当前产品策略：
- 基础主题默认可直接使用
- 其他主题当前可直接切换，但在产品定义上已按“可扩展付费主题”结构设计
- 当前尚未接入支付、权益校验和主题商城

当前主题设计边界：
- 主题目前主要影响颜色、渐变、卡片层次与强调色
- 尚未覆盖图标包、插画包、排版模板、动态效果模板

## 6. 当前提醒规则细则

### 6.1 目标完成后的提醒处理
- 用户当日打卡后，会先清掉该目标已有提醒，再按最新状态重新调度
- 已暂停目标不会继续注册提醒
- 每周次数目标在当周达标后，不再继续注册当周剩余提醒
- 对“每日 / 每周固定日 / 每周次数”目标，若当天已完成，则重建窗口时默认不再包含今天，因此当天剩余监督提醒会被取消

### 6.2 每周次数目标规则
- “今天是否待做”不再固定为 true
- 当前周累计完成次数达到目标值后，该目标本周视为已达标
- 达标后不会继续在首页“待完成”中出现
- 若今天完成并刚好达标，仍会在首页“已完成”中保留展示

### 6.3 前后台展示策略
- 前台通知展示由 `UNUserNotificationCenterDelegate` 接管
- 当前前台收到通知时允许 `.banner`、`.list`、`.sound`、`.badge`
- 普通通知音仍然是单次触发，不是持续响铃
- 持续响铃依赖 AlarmKit 闹钟式能力
- App 首次进入前台激活时，会执行一次启动期提醒全量调度，并同步校验当前倒计时会话是否仍然有效
- 后续普通前后台切换不再自动全量重建目标提醒；如需手动重建，可使用设置页“重新同步所有提醒”

## 7. 数据模型与持久化

### 7.1 当前持久化实现
当前并未使用 SwiftData、Core Data 或 SQLite，而是采用本地 JSON 持久化。

实现方式：
- 数据保存在 App `Application Support/LazyButNot/goals.json`
- `GoalStore` 负责加载、修改与落盘
- 编码解码使用 `Codable`
- 日期采用 ISO8601 编码策略

## 9. 验收口径

当前版本应满足以下验收标准：
- 可以创建、编辑、删除目标
- 可以对目标执行完成或保底完成打卡
- 重启 App 后目标和历史记录仍然存在
- 首页能正确区分今日待完成与已完成
- 首页待完成和已完成卡片的非按钮区域均可进入目标详情
- 首页未完成卡片展示操作按钮，已完成卡片展示紧凑状态徽标，不再显示重复操作按钮
- 每周次数目标达标后不再继续产生当周提醒
- 可以在设置页切换主题，且切换后首页/目标页/统计页/设置页样式同步变化
- 目标提醒与监督提醒按当前固定日期窗口成功生成
- 当天完成目标后，当天剩余监督提醒不会继续触发
- 设置页能展示通知/闹钟权限状态
- iOS 26+ 已授权设备上可创建倒计时闹钟
- 倒计时闹钟创建成功后可进入专注详情页
- 专注详情页可取消当前倒计时并返回首页
- 倒计时闹钟具备 Live Activity 展示与回跳能力
